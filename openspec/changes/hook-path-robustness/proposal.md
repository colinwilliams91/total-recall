## Why

The post-commit hook template at `cmd/tr/main.go:101-111` (`postCommitHookScriptTmpl`) bakes the absolute binary path via `os.Executable()` capture into the hook script at `tr init`/`tr repo` time via two `%s` insertions — one for the PowerShell branch, one for the sh fallback. If the user later moves or rebuilds the binary (e.g. `go install` puts a fresh binary at `$GOPATH/bin/tr` after a code change), the installed post-commit hook keeps trying the OLD path and fails silently (or loudly, in a way the user can't trace back). With the Y5 install docs set to advise every user to keep `tr` on PATH, the simpler "rely on PATH" approach (Issue 5 Decision 5.B) is the right balance — Y4 strips the path-baking logic from the post-commit hook template.

## What Changes

- Modify `postCommitHookScriptTmpl` in `cmd/tr/main.go:101-111` to drop both `%s` insertions. The template becomes a static string:
```sh
#!/bin/sh
# total-recall managed
# Surfaces a recall question after each successful commit.
# Generated by 'tr repo'. Run 'tr repo' again to reinstall.
if command -v powershell.exe >/dev/null 2>&1; then
    powershell.exe -NoProfile -ExecutionPolicy Bypass -Command "tr ask"
    exit $?
fi

exec tr ask
```
- The `buildPostCommitHookScript(selfPath string)` function (`cmd/tr/main.go:113-117`) becomes either a no-arg function returning the constant template, or is deleted entirely with the constant template inlined at the call site. The implementation decision is in tasks.md and defaults to "delete the function; inline the constant."
- The `selfPath, err := os.Executable()` capture at `cmd/tr/main.go:259-262` is deleted. The fallback `if err != nil { selfPath = "total-recall" }` and the `buildPostCommitHookScript(selfPath)` call site are deleted.
- Update `cmd/tr/main_test.go`'s `buildPostCommitHookScript` tests: the tests previously asserted on the `%s`-substituted output. After Y4, the test asserts on the static template string (no substitutions). The tests become simpler — they assert on a literal `strings.Contains` or `==` match against the constant.
- The Y3 phase includes an optional "Y4 fold-in" task (already documented in `openspec/changes/init-repo-split/tasks.md` Section 7). If Y3 and Y4 are implemented jointly, this Y4 change proposal becomes redundant — the Y3 task 7 subsumes it. If Y3 ships without folding Y4 in, this Y4 change is implemented as its own small phase.

**Non-changes (deliberate):**

- No `command -v tr` fallback to absolute path (Issue 5 Option 5.A rejected). The hook script relies entirely on PATH — the simplest template.
- No PATH-detection logic in the hook script itself. Y5's `tr init` startup detection handles that — it warns the user if `tr` is not on PATH before they ever run `tr repo` and install a hook that would fail.
- The sentinel `# total-recall managed` and the `Generated by 'tr init'`/`Generated by 'tr repo'` content lines stay (consistent with Y2's sentinel decision and Y3's command rename). The "Generated by" line becomes `Generated by 'tr repo'` per Y3's split.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `post-commit-hook`: the spec requirement "Post-commit hook calls tr ask" is amended to specify `exec tr ask` without the `$(which tr)` indirection. The Y3 spec delta had `exec "$(which tr)" ask`; Y4 strips the `$(which ...)` indirection to call `tr` directly via PATH. (Both Y3 and Y4 amend the same requirement; Y4's spec delta is layered over Y3's — implementer should refer to the latest delta at archive time.)

## Impact

- **Code edited:** `cmd/tr/main.go` — `postCommitHookScriptTmpl` constant (lines 101-111) reduced to a static string; `buildPostCommitHookScript` function (lines 113-117) either deleted or simplified to return the constant; the `os.Executable()` capture and fallback (lines 259-262) deleted; `buildPostCommitHookScript(selfPath)` call at line 263 changed to inline the template.
- **Tests updated:** `cmd/tr/main_test.go` — `buildPostCommitHookScript` (or whatever the tests call after deletion) tests updated to assert on the static template. No new tests needed.
- **Layers affected:** the **binary** layer — the post-commit hook TEMPLATE baked-in at compile time changes. The **git hooks** layer — newly installed post-commit hooks (via `tr repo`) reference `tr` by name rather than by baked absolute path. Existing installed post-commit hooks (from prior installs) are overwritten on the next `tr repo` because the installer detects the unchanged `# total-recall managed` sentinel.
- **Dependencies:** none added or removed.
