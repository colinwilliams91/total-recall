> Informed by white papers, labs and datapoints

## 1. The biggest learning loss is "Debugging"

The largest performance gap between AI-assisted and non-AI developers appears in:
- debugging
- reasoning about failures
- understanding runtime behavior

--AI generated code is error prone at the systems level, so being able to debug will always be an invaluable skill.

## 2. Cognitive Engagement Preserves Learning

The three “high-scoring” AI interaction patterns:
- Conceptual Inquiry
- Hybrid Code + Explanation
- Generation Then Comprehension

## 3. Error encountering is essential for learning

Recreating the learning value of encountering errors  
without necessarily requiring the user to manually suffer through them.

## 4. Pure Delegation Produces The Worst Outcomes

In Hanwen-Shen and Tamkin's [How AI Impacts Skill Formation](https://arxiv.org/pdf/2601.20245) administered a 27-question quiz testing engineers' ability to comprehend and retain necessary skills to use a new coding library after implementing tasks with it with the assistance of AI. Focus groups who did pure "AI delegation" scored lowest of all focus groups averaged at 39% scores. Focus groups who used "Generation-Then-Comprehension", the highest scoring group, generated code with AI and then queried afterwards for comprehension--averaged scores at 86%.  

## 5. Convert AI-assisted implementation into durable understanding

Generation-then-comprehension is a data-backed pattern for enhanced skill development. In unfamiliar domains engineer's are slowed down cognitively by virtue of learning. Coupling 4 seconds of inquiry with AI assistance may actually boost skill development over manually reading documentation and implementing code.

## 6. Contextual Retrieval Practice

[How AI Impacts Skill Formation](https://arxiv.org/pdf/2601.20245) and Anthropic Research Team write up on it: [How AI assistance impacts the formation of coding skills](https://www.anthropic.com/research/AI-assistance-coding-skills) repeatedly emphasize:

- on-the-job learning
- workflow integration
- real coding tasks
- contextual learning

—not isolated education.

This should happen:

- during commits
- during pushes
- during actual implementation work

—not in separate educational portals.
